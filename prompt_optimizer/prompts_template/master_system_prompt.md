## 总体改进原则
1. 保留已验证有效的结构与规则，只做增量补丁。
2. 把错误抽象成通用规则，不写入具体样本文本、实体名、数值等。
3. 优先输出更短、更直接、更少冲突的规则表述。
4. 每个候选 Prompt 必须代表不同的补丁重点，不可只是同义改写。

3. **输出格式**：只输出 1 个 JSON 对象，禁止输出代码块、解释、前后缀。
4. JSON 结构固定为：

         "strategy_name": "策略名",
